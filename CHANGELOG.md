# Changelog

## 1.0.0-rc.3 — 2026-07-21

This candidate supersedes the withdrawn RC.2 artifacts. It was developed under the pinned `gpt-pro-controlled-delivery/0.3` workflow using RED-first slices and cumulative sealed checkpoints.

### External-review kill-shot repairs

- makes unseen consequential English and Chinese requests fail closed while preserving read-only near misses and explicit bounded local implementation;
- validates Deep Start Card facts, assumptions, decisions, success verifiers, counterexample challenge, durable user-confirmation source, Permit expiry, revocation, lease break, and route-bound activation;
- requires every material product change, required output, and exit criterion to be covered by predeclared checks against final bytes and approved scope;
- prevents explicit Success Capture labels from raising routine work and derives candidate sequence, decisive conditions, command, artifacts, and evidence IDs from real receipts;
- preserves complete `.agents`, `.claude`, `.vibetether`, host instructions, routes, locks, and modified assets across migration and conflict-aware rollback;
- provides transactional RC upgrade/rollback and an offline verified local launcher cache;
- persists repository/worktree UUIDs independently from mutable Git locator paths and keeps Provider/Evidence subprocess environments allowlisted.

### Evidence status

Local Linux verification is supporting evidence only. Exact live v0.6.3 acquisition, the Ubuntu/Windows Node 20/24 matrix, and independent external review remain promotion gates. This artifact is not released or independently accepted.

## 1.0.0-rc.2 — 2026-07-20

This candidate supersedes the withdrawn `1.0.0-rc.1` artifact. It turns the independent Sol audit findings into release-blocking regression tests instead of treating package integrity or Linux-only self-tests as product conformance.

### Core gate repairs

- adds the separate `vibe-tether-deep` Entry Skill, Start Card, expiring user-confirmed Implementation Permit, permit invalidation, and permit consumption;
- blocks code-write when task classification requires a user decision unless a bounded decision reason or valid Deep Permit exists;
- makes completion require a satisfied route, successful final evidence, a validated route output contract, resolved Truth reconciliation, and Success Capture disposition;
- binds evidence to before/after execution snapshots and rejects product-byte changes after the final successful evidence;
- validates path-like required outputs by existence, safe path, and content digest;
- records route additions to required outputs and exit evidence as a union rather than replacing them with defaults.

### Compatibility and worktree repairs

- parses the real 0.6.3 canonical Truth sections, including Host bootstrap and Control-plane pointers, without treating those bootstrap entries as project authority;
- migrates section-based and metadata-based 0.6.3 Intent Contracts into the 1.0 format;
- adds a sanitized fixture preserving the structure of a real installed 0.6.3 control plane, plus byte-preserving rollback checks;
- preserves legacy confirmed Truth, converts unconfirmed manifest sources to candidates, and downgrades legacy proven Experience to provisional;
- keeps rollback from overwriting assets modified after migration;
- normalizes tracked text authority and generated entry assets across LF/CRLF worktree checkouts and adds `.gitattributes` policy;
- keeps stable VibeTether worktree IDs while Git administrative paths remain current locators.

### Beginner and routing repairs

- implements actual TTY-guided initialization with an injectable prompt adapter for tests;
- recognizes realistic vague English and Chinese feature requests and prevents silent implementation;
- applies Truth role, phase, operation, module, and path metadata when selecting applicable authority;
- keeps stable Truth handles and reports total, returned, and omitted counts;
- classifies reusable operational successes by evidence, non-obvious conditions, and rediscovery value, and records structured candidate sequences and reasons.

### Provider and security hardening

- retains the rich cold `standard`, `extended`, `web`, and `production` packs while exposing only the adaptive and deep VibeTether entry Skills by default;
- enforces active-route checks and a minimal environment allowlist for Provider script execution;
- retains declaration-only license evidence as metadata but excludes those Provider bytes from the distributable archive; equivalent cold capabilities are supplied from sources with complete retained license text;
- retains content-addressed Provider cache limits, corruption quarantine, path protection, and recoverable file replacement.

### Release status

This is still a release candidate. Local verification does not substitute for the configured Ubuntu/Windows, Node 20/24 matrix or independent review. Do not promote it to final 1.0 until those gates pass.

## 1.0.0-rc.1 — withdrawn

The prior RC package was installable and internally consistent on its local Linux test set, but independent review found missing deep/user-decision gates, incomplete completion enforcement, real 0.6.3 migration incompatibility, CRLF worktree divergence, absent guided setup, and incomplete Truth applicability. It must not be released or used to replace 0.6.3.
