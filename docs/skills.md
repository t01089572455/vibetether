# Skill broker and Provider lifecycle

## Catalog versus context

Provider cards and content-addressed objects live in the user cache. Search and inspect return bounded metadata. A project lock pins approved Provider bytes. Routing checks phase, capability, signals, host, OS, permissions, channel, pins, project preferences, failure history, measured quality, and context cost.

The shortlist contains at most three candidates. Exactly one primary Provider controls a step. Activation verifies immutable content and creates a scope envelope; undeclared resources, unsafe paths, and symlinks are rejected.

## Profiles, bundles, and permissions

`core`, `standard`, and `extended` select reviewed eligibility sets. The optional `web` and `production` bundles add domain-specific Providers. Eligibility never grants network, external-write, or code-write permission. A Provider remains below platform safety, user direction, confirmed Truth, the current slice, and the evidence contract.

No Provider is downloaded during an active route. Missing optional capability uses a verified built-in fallback; permission-sensitive work stops when no eligible Provider exists.

## Custom routes

The tracked `.vibetether/routes.json` may select a primary or alternative Provider for observable signals, or add an overlay. Routes are validated against known phases, public capabilities, Providers, and permissions. An overlay may add required outputs and exit evidence; it cannot remove platform or Outcome requirements.

Upgrade and rollback preserve user routes, project preferences, disabled Providers, and pins byte-for-byte unless an explicit migration contract says otherwise.

## Trust and exposure

Imported Providers begin `experimental`. Measured trigger quality and output gain are required before beta or stable promotion. Two recorded project failures remove a Provider from automatic routing without deleting its history.

Native host exposure is opt-in. Normal discovery contains the small `vibe-tether` and `vibe-tether-deep` entries; community Providers remain cold. A maximum of four project Hot Skills may be named. Modified exposed bytes are never overwritten or removed automatically.

Release and tag labels are provenance hints, not immutable identity. The registry records the resolved commit and normalized content digest; declaration-only license evidence remains marked for legal review.
