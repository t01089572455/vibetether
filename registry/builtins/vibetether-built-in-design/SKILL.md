# vibetether-built-in-design

Present bounded alternatives and trade-offs without propagating an unapproved direction.

For user-visible UI work, keep the ordered contract explicit: UI discovery, product UX contract, reference intake, design contract and tokens, then one representative golden screen and owner approval. Do not begin implementation or propagate a visual direction before that approval.

Stay inside the VibeTether scope envelope. Return the capability outputs and fresh exit evidence. Never override confirmed project truth, the current slice, or a user decision gate.
